Native AST Implementation - IN PROGRESS

Instructions

Please see the details for each step below. The steps are then highlighted through comments directly in the embedded code underneath.

Step 1: you will need to create the APN object and load the AppNexus JavaScript library. This part of the code can be copy-pasted directly into your web page - no additional developer changes are required. This usually goes in the <head> of the of the page, but if you don't have access to that, you can put the entire tag in one code block in the <body>.

Step 2: this is the continuation of the above step. Here you will need to make some adjustments in setting the global page options: you will need to set to correct member ID (the one in the example is 958, but this will need to be replaced with your own member seat) and define the page-wide targeting parameters (there are none set in this example, but if you want to set some, please see the documentation linked after the instructions).

Step 3: now we will define the tag. In the example below you can see the code used to define the tag implemented on this page. Here you will also define the native requirements the seller/publisher has for what the creative should contain (title, image etc).

Step 4: right after the tag is defined, it will need to be loaded. This is the last part of the code which is usually found in the <head> section.

Step 5: now we will implement the script used to render native in the <body> part of the webpage, where we want it to appear; this is self-contained and will render in-stream. You can modify $ad.innerHTML with your own HTML and CSS to match the look and feel of your content.

Step 6: finally, it's time to load the element in order to show the ad.

You can find more detailed information about the implementation here: AppNexus Wiki - Set up Placements with AST.

For further AST API references, you can also refer to the following documentation: AppNexus Wiki - AST API Reference.

Additionally, for native, we also support native video through AST so feel free to check Native Video with AST to see how this can be configured and what kind of options are available.

Further things to note:

  • Setting allowedFormats on AST is optional; if it's set, it will override the media types assigned on the placements' settings in Console.
  • You can add multiple sizes and media types on the tags so that, for example, it will allow banner to serve on the same placement, if there's no native demand available. However, this is not supported by all external bidders - please contact your AppNexus Account Manager for details on which bidders support this.
  • Even if the above is not supported, it's still good to add both native and banner media types, with native as first option, since if it's not supported, then only the first option will be taken into consideration.

Finally, regularly checking on the AST release history is encouaged so that you can see the latest releases and adjust the code accordingly if needed (due to breaking changes, for example).
